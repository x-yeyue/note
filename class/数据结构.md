# 数据结构的基本概念和算法

## 数据结构的基本概念

### 数据

**数据** $Data$，所有能输入到计算机中的描述客观事物的符号。

### 数据元素

**数据元素 $Data \; Element$(节点、记录)**， 数据的基本单位。

### 数据项

**数据项 $Data \; Item$(域 $Domain$)** 表示有独立含义的 *数据最小单位*。
- 若干个数据项构成一个数据元素。
- 数据项是不可分割的最小单位。

### 数据对象

**数据对象 $Data \; Object$(数据元素类 $Data \; Element \; Class$)**，同质的数据元素的集合。
- 数据元素是数据对象或数据元素的一个实例。

### 数据结构

**数据结构 $Data \; Structure$**，通常包括 **数据元素之间的逻辑结构**、**数据元素在计算机内的存储结构**、**定义在这种结构上的一组操作**。

**基本数据结构**：
	1. 线性结构(1:1)
	2. 树结构(1:N)
	3. 图形结构、网状结构(M:N)
	4. 集合结构 

**基本要素**: 数据的集合、关系的集合。
	可以表示为一个二元组:		$$Data\_Structure = (D, R)$$
	其中 $D$ 为数据元素的有限集，$R$ 为 $D$ 上的有限集。

**分类**: 
	*根据数据之间存在的关系*: 线性结构、非线性结构。
	*根据视点的不同*: 数据的逻辑结构、数据的物理(存储)结构。
	*根据在计算机中的存储表示*: 顺序结构、非顺序结构(链式存储)、散列存储($Hash$)。

### 抽象数据类型

数据类型按“值”的不同特性可分为: 非结构的原子类型、结构类型。
	原子类型是不可分解的。如 整型、实型、字符型等。
	结构类型是可分解的。

**抽象数据类型($Abstract \; Data  \; Type, ADP$)**，是一个数学模型以及在该类型上的一组操作。

抽象数据类型的 3 **要素**: 元素、关系、操作。

抽象数据类型的**描述**包括: 数据类型的名称、数据的集合、数据之间的关系、操作的集合。

```
ADT 抽象数据类型名称{
	数据对象: <数据类型的定义>
	数据关系: <数据逻辑关系的定义>
	基本操作集:
		操作名 1:
			<基本操作 1 的初始条件描述>
			<基本操作 1 的操作结果描述>
		·
		·
		·
		操作名 n:
			<基本操作 n 的初始条件描述>
			<基本操作 n 的操作结果描述>
}ADT 抽象数据类型名称
```

从抽象数据类型的角度看，数据结构是抽象数据类型的物理实现。

数据结构包括 逻辑结构、存储结构、数据运算。
	**逻辑结构**:
		从数据之间的逻辑关系上描述数据。
	**存储结构**: 
		数据元素及其关系在计算机存储器内的表示。
	**数据运算**: 
		对数据施加的操作。
## 算法的执行效率及其度量

算法是指对特定问题求解步骤的一种描述。
通常是以算法执行所耗费 *时间* 和所占的 *空间* 来判断一个算法的优劣。

### 算法的特性

- **有穷性**: 算法是由若干条指令组成的又穷序列，总在执行若干次后结束。
- **确定性**: 每条语句由确定的含义，无歧义。
- **可行性**: 算法在当前环境下可以通过有限次运算实现。
- **输入**: 有零个或多个输入。
- **输出**: 一个或多个输入。

### 好算法的标准

- **正确性**: 算法能够满足具体问题的需求。
- **易读性**: 算法遵循标识符命名规则，简洁、易懂，注释语句恰当、适量。
- **健壮性**: 算法对非法数据及操作有较好的反应和处理。
- **高效性**: 算法运行效率高。
- **低存储性**: 指算法所需要的存储空间低。

### 时间复杂度

算法所需要的空间，一般将算法基本运算的执行次数作为时间复杂度的衡量标准。
