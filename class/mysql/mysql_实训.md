# 关系模式

**用户**(用户ID, 用户名, 密码, 手机号, 邮箱, 用户类型, 注册时间)
**课程**(课程ID, 课程名称, 课程简介, 讲师ID, 视频地址)
**问题**(问题ID, 课程ID, 提问用户ID, 问题类型, 内容)
**编程题目**(题目ID, 所属课程ID, 题干, 示例输入输出)
**回复记录**(回复ID, 问题ID, 回复用户ID, 回复内容)
**学习记录**(学习记录ID, 用户ID, 课程ID, 学习开始时间, 完成状态)
**作答**(用户ID, 题目ID, 作答时间, 正确率, 完成状态)
	*外键*: 
		用户ID 引用 用户(用户ID)
		题目ID 引用 编程题目(题目ID)
	*描述*: 用户和编程题目间的二元关系。

**关系模式说明**: 
	*用户*: 记录用户的基本信息，通过“用户类型”区分普通用户和老师。
	*课程*: 记录课程的基本信息。
	*问题*: 记录问题的基本信息。
	*编程题目*: 记录编程题目的基本信息。
	*回复记录*: 记录用户的回复记录的基本信息。
	*学习记录*: 记录用户的学习记录的基本信息。
	*作答*: 表示用户的编程题目作答情况(多对多)。
